# 快捷键最终方案

## 当前快捷键配置

经过多次测试和调整，最终确定的快捷键方案：

- **F10**: 开始/停止录音
- **F11**: 切换工作模式

## 为什么选择 F10 和 F11

### 历史尝试

1. **F4** - 被系统占用
2. **[ 和 ]** - 作为输入键，不适合
3. **Tab+[ 和 Tab+]** - Tab 是导航键，会切换焦点
4. **Ctrl+Shift+[ 和 Ctrl+Shift+]** - 注册失败
5. **F8 和 F9** - 被其他应用占用
6. **Alt+[ 和 Alt+]** - 无响应，可能被占用
7. **Ctrl+Alt+R 和 Ctrl+Alt+M** - 注册失败，被占用
8. **F10 和 F11** - ✅ **成功！**

### F10/F11 的优势

1. **注册成功**: 经测试，这两个键在系统中未被占用
2. **位置相邻**: 两个功能键紧挨着，便于操作
3. **易记**: F10=录音(Record)，F11=模式(Mode)
4. **标准键**: 不需要组合键，单键触发更可靠
5. **全局有效**: 即使应用在后台也能响应

## 工作模式

按 F11 循环切换三种模式：

1. **转写模式** (Transcribe)
   - 仅识别中文，直接输出
   - 适合中文输入场景

2. **翻译模式** (Translate)
   - 识别中文后翻译成英文
   - 仅输出英文翻译结果
   - 使用 GPT-OSS-20B 优化为书面语

3. **双语模式** (Dual)
   - 同时显示中文和英文
   - 格式：
     ```
     中文: [原文]

     English: [翻译]
     ```

## 使用流程

1. 按 **F10** 开始录音（系统托盘图标会显示 "Recording"）
2. 说出您要转写/翻译的内容
3. 按 **F10** 停止录音
4. 系统自动处理并复制到剪贴板
5. 按 **F11** 可随时切换模式

## 技术细节

### 防抖机制
- 800ms 防抖保护
- 防止快捷键重复触发
- 同时使用布尔标志和时间戳检测

### 日志输出
启动时会显示：
```
Record hotkey registered: F10 success: true
Mode toggle hotkey registered: F11 success: true
```

如果看到 `success: false`，说明快捷键被占用。

### 配置文件
快捷键保存在 `config.json`：
```json
{
  "hotkey": "F10",
  "modeToggleHotkey": "F11",
  "mode": "transcribe",
  "translationEnabled": true,
  "translationServerUrl": "http://192.168.2.2:1234",
  "translationModel": "gpt-oss-20b",
  "translationStyle": "professional"
}
```

## 自定义快捷键

如果 F10/F11 在您的系统上不可用，可以在设置界面修改：

1. 右键系统托盘图标
2. 选择 "Show Settings"
3. 修改"录音热键"和"模式切换热键"
4. 点击"保存设置"

支持的快捷键格式：
- 单键: `F1`-`F12`, `A`-`Z`, `0`-`9`
- 组合键: `Ctrl+A`, `Alt+B`, `Shift+C`
- 多修饰键: `Ctrl+Shift+A`, `Ctrl+Alt+B`

## 故障排除

### Q: 快捷键不响应？
A:
1. 检查启动日志，确认 `success: true`
2. 确保应用正在运行（托盘图标存在）
3. 尝试重启应用
4. 尝试更换其他快捷键

### Q: 如何验证快捷键工作？
A:
1. 按 F10，观察托盘图标提示是否变为 "Recording"
2. 打开设置界面，按 F11，观察"当前模式"状态栏变化

### Q: 录音立即停止？
A:
- 已添加 800ms 防抖保护
- 如仍有问题，可能是系统兼容性问题
